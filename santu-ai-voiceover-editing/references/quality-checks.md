# 交付检查

## 音频

- [ ] 默认使用本地 MiniMax 中国站 CLI `minimax-tts`，或明确记录用户提供的成品音频。
- [ ] TTS 使用 `--voice-id SantuVoice20260618B`。
- [ ] TTS 使用 HD 模型 `--model speech-2.8-hd`。
- [ ] 音频文件路径写入时间线。
- [ ] 字幕时间基于真实音频，不是纯文本估算。
- [ ] 重点句有自然停顿，不一路念平。

## 画面

- [ ] 画幅是 `1080x1920`。
- [ ] 背景是黑色或近黑色。
- [ ] 上方是黑底简笔画图片，不是白底图。
- [ ] 插画显示框为等比例轻缩，左右至少各留 `80px`，顶部至少留 `60px`，不要贴边。
- [ ] 字幕在插画和头像之间。
- [ ] 字幕背后有黑色/近黑柔边安全带，即使插画线条经过也不影响读字。
- [ ] 头像圆形裁切，位于中间偏下。
- [ ] 头像左右有固定动态声波。
- [ ] 声波不压字幕、不挡头像主体。
- [ ] 字幕底部离头像顶部和声波顶部至少 `20px`。

## 插画

- [ ] 简笔画素材由 gpt-image2 生成图片文件。
- [ ] 每张插画已经保存到 `public/media/images/` 或等价工程目录。
- [ ] 每张插画已用 `file` / `sips` / 图片查看工具确认可读、尺寸正确。
- [ ] `asset-manifest.json` 和 `timeline.json` 记录真实图片尺寸，不假写 `1080x1920`。
- [ ] `timeline.json` 同时记录源尺寸和安全区显示尺寸；显示缩放通常为 `0.90-0.95`。
- [ ] 正常生成的图片没有被剪辑层缩小到小框里；只做平台安全区需要的等比例轻缩。
- [ ] 代码没有生成简笔画主体。
- [ ] 时间线没有引用不存在的图片路径。
- [ ] 每张图只表达一个核心意思。
- [ ] 图像手机端 1 秒内能看懂。
- [ ] 图内文字少，避免模型写错字；关键信息尽量交给字幕。

## 字幕

- [ ] 每屏最多 2 行。
- [ ] 字幕按逗号、句号、问号、冒号等停顿拆成短段逐段显示。
- [ ] 字幕没有贴底。
- [ ] 字幕没有挡住插画、头像或声波。
- [ ] 关键词高亮克制。

## 交付

- [ ] 有 TTS 音频或 TTS 任务清单。
- [ ] 有 `timeline.json`。
- [ ] 有 gpt-image2 prompt 列表。
- [ ] 有插画素材目录。
- [ ] 有 Remotion 工程。
- [ ] 有最终 MP4。
- [ ] 有关键帧检查图。
- [ ] 已从最终 MP4 或最终 Remotion 输出抽帧检查，不只看聊天预览或开发预览。
- [ ] 已用 `ffprobe` 确认输出为 `1080x1920`、`30fps`，并确认文件真实写入 `out/`。
