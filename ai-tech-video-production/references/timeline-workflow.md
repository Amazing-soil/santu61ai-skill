# Timeline Workflow

## Principle

先切真实时间线，再匹配素材。不要按文案主观估算素材时间。

## Steps

1. 读取用户给的口播文案。
2. 读取或检查口播视频。
3. 如果有字幕文件，优先用字幕时间戳。
4. 如果只有烧录字幕或原视频，先抽帧/转写/人工校准关键句时间点。
5. 标记开头钩子段和结尾收束段。默认前 `3 秒` 与最后 `3 秒` 使用居中强化出镜；如果真实句子更短，按真实边界缩短。
6. 按 `3-5 秒` 一个视觉单元组织段落。
7. 将每段标成：
   - `video`：HTML/Anime/实操 MP4
   - `image`：GPT Image 或静态图
   - `none`：不加素材
8. 对每段填写：
   - `id`
   - `start`
   - `end`
   - `kind`
   - `asset`
   - `assetDuration`
   - `note`
   - `enabled`

## Matching Rules

- 有真实操作素材：优先使用真实操作素材。
- 没有真实操作素材：再考虑 HTML/Anime 或 GPT Image。
- 同一句口播太短：不要强行塞完整动效。
- 连续观点段：可以让素材桥接，不要中间闪回口播底图。
- 每个素材只服务一个核心意思。

## Handoff Notes

给剪辑师的说明必须写清：

- 哪句口播对应哪个素材。
- 素材是全屏、上半屏、右侧卡片还是背景。
- 真人 PIP 使用 `introHero`、`bodyCorner` 还是 `outroHero`。
- 居中强化出镜期间，人物周围保留什么背景素材、主标题和短标签。
- 是否需要加速/放慢。
- 是否可以删掉不用。
