# Timeline Workflow

## Principle

先切真实时间线，再匹配素材。不要按文案主观估算素材时间。

## Steps

1. 读取用户给的口播文案，把它作为语义参考，不作为最终字幕底稿。
2. 读取或检查口播视频。
3. 先从视频音频做 ASR/转写；如果有字幕文件，可优先借用时间戳，但字幕文字必须和真实语音口播核对。
4. 如果只有烧录字幕或原视频，先抽帧/转写/人工校准关键句时间点；文案和口播不一致时，以口播为准。
5. 标记开头钩子段和结尾收束段。默认前 `3 秒` 与最后 `3 秒` 使用居中强化出镜；如果真实句子更短，按真实边界缩短。
6. 用户要求字幕时，生成独立字幕轨：按真实口播自然句、语义短句和停顿切分，每条尽量单行，不机械按字数断行。
7. 按 `3-5 秒` 一个视觉单元组织段落。
8. 将每段标成：
   - `video`：HTML/Anime/实操 MP4
   - `image`：GPT Image 或静态图
   - `none`：不加素材
9. 对每段填写：
   - `id`
   - `start`
   - `end`
   - `kind`
   - `asset`
   - `assetDuration`
   - `note`
   - `enabled`

## Matching Rules

- 有真实操作素材：优先使用真实操作素材。
- 没有真实操作素材：再考虑 HTML/Anime 或 GPT Image。
- 同一句口播太短：不要强行塞完整动效。
- 连续观点段：可以让素材桥接，不要中间闪回口播底图。
- 每个素材只服务一个核心意思。
- 字幕匹配优先听真实音频；用户文案、旧字幕、标题稿只用于理解语义，不反向纠正真实口述。
- 如果口播里有口误、重复、临场改词，字幕保留真实表达为主；只在明显错听或无意义语气词影响阅读时做轻微清理。

## Handoff Notes

给剪辑师的说明必须写清：

- 哪句口播对应哪个素材。
- 素材是全屏、上半屏、右侧卡片还是背景。
- 真人 PIP 使用 `introHero`、`bodyBottomCenter` 还是 `outroHero`。
- 居中强化出镜期间，人物周围保留什么背景素材、主标题和短标签。
- 如果生成字幕，写清字幕来自真实口播 ASR/人工校准，不是直接复制用户文案；同时写清字幕位置、字体栈、字重、阴影和动态字号规则。
- 是否需要加速/放慢。
- 是否可以删掉不用。
