# Quality Checks

## Layout

- [ ] 非图片素材（HTML/Anime.js、文字信息图、流程卡、图表、代码/窗口模拟）没有贴近屏幕上沿。
- [ ] `Y=0-260` 只放背景氛围；首个核心文字/卡片/图标组/流程节点顶部不进入顶部安全区，建议 `Y>=280`。
- [ ] 非图片素材核心可读信息主要位于 `Y=280-1080`。
- [ ] 烧录字幕位于中间偏下安全区，而不是贴正下方。
- [ ] 字幕不遮挡右下真人 PIP，PIP 也不压字幕。
- [ ] 核心文字和关键图形不进入中间偏下字幕区。
- [ ] 右下真人 PIP 没遮住主素材。
- [ ] 如果不烧录字幕，中间偏下区域可给后期字幕使用。
- [ ] PIP 人像没有被拉伸。
- [ ] PIP 是原口播视频，不是素材。

## Timing

- [ ] 素材出现时间与口播语义对齐。
- [ ] 每 `30 秒` 至少有 `1` 个 GPT Image / AI 图片素材，最好 `1-2` 个；全实操录屏窗口可例外。
- [ ] 太短段落没有硬塞播不完的素材。
- [ ] MP4 动效已按段落时长变速。
- [ ] 每 `3-5 秒` 有视觉变化，但不为变化而变化。

## Transition

- [ ] 相邻素材没有露出口播底图。
- [ ] 桥接空档不会闪屏。
- [ ] 覆盖式淡入时上一段素材保持不透明。

## Content

- [ ] 没有未核验新闻截图或虚假数据。
- [ ] 实操段没有被概念图盖住。
- [ ] 不是整条视频都用文字信息图；已混合 GPT Image、动效、实操/截图等素材类型。
- [ ] GPT Image 图片不包含字幕、标题、金句、品牌 Logo、真实 UI 或假冒截图。
- [ ] 文字短句化，适合中文观众快速读懂。
- [ ] 术语有必要时用用户语言解释。

## Delivery

- [ ] 有最终 MP4。
- [ ] 有可编辑工程或清晰的剪辑师交付表。
- [ ] 有关键帧检查图。
- [ ] 有素材目录说明。
- [ ] 有快速修改入口，例如 `src/timeline.json`。
